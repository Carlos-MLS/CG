COMPILAR (CMAKE):
  $ cd Trabalho && mkdir -p build && cd build
  $ cmake .. -B . && make

GERAR PRIMITIVAS:
  $ cd build
  $ ./generator plane 5 10 plane.3d
  $ ./generator box 2 3 4 5 box.3d
  $ ./generator sphere 1 20 20 sphere.3d
  $ ./generator cone 1 2 20 10 cone.3d

VER NO ENGINE (OpenGL):
  $ ./engine config.xml
  O config.xml define a janela, camera e que modelos carregar.
  Controlos:
    WASD  - rodar camera
    Q/E   - zoom in/out
    C     - alternar camera fixa/orbital
    1     - modo solido
    2     - modo wireframe
    3     - modo pontos
    ESC   - sair

FORMATO .3D:
  Linha 1: número de vértices
  Restantes: x y z (um vértice por linha)

FORMATO XML (config):
  <world>
    <window width="512" height="512" />
    <camera>
      <position x="3" y="2" z="1" />
      <lookAt x="0" y="0" z="0" />
      <up x="0" y="1" z="0" />
      <projection fov="60" near="1" far="1000" />
    </camera>
    <group>
      <models>
        <model file="plane.3d" />
      </models>
    </group>
  </world>

IMPLEMENTADO:
  - generator/primitives/plane.h → gerarPlane()
  - generator/primitives/box.h → gerarBox()
  - generator/primitives/sphere.h → gerarSphere()
  - generator/primitives/cone.h → gerarCone()
  - engine/main.cpp → engine OpenGL com parser XML

ESTRUTURA:
  generator/
    main.cpp              - programa que gera os .3d
    primitives/           - plane, box, sphere, cone
  engine/
    main.cpp              - engine OpenGL/GLUT
    parser/xmlparser.h    - le o XML de config
    model/model.h         - carrega ficheiros .3d
  utils/
    point.h               - struct Point3D
    fileWriter.h          - escreve vertices para ficheiro

CLION:
  File → Open → 'Trabalho/' → Build (Ctrl+F9)

NOTES:
  - Plane: XZ centrado (y=0)
  - Box/Sphere/Cone: centrados na origem
  - Cone: base em XZ, vértice em (0,height,0)
  - Vértices em sentido anti-horário
